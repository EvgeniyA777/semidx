# Working Note: Tree-sitter Coverage by Supported Language

Краткая dated-выжимка по тому, для каких поддерживаемых языков в рантайме уже есть tree-sitter, а для каких его еще нет.

## Итог

Поддерживаемые языки сейчас такие: `clojure`, `java`, `elixir`, `python`, `typescript`, `lua`.

Tree-sitter реально интегрирован только для:

- `clojure`
- `java`
- `typescript`

Tree-sitter еще не используется для:

- `elixir`
- `python`
- `lua`

## Нюанс по умолчанию

Даже там, где tree-sitter уже есть, он не везде является дефолтом:

- `clojure` по умолчанию идет через `:clj-kondo`
- `java` по умолчанию идет через `:regex`
- `typescript` по умолчанию идет через `:regex`

## Опорные точки в коде

- Список поддерживаемых языков: `src/semidx/runtime/language_activation.clj`
- Общий tree-sitter wiring и grammar-path lookup: `src/semidx/runtime/adapters.clj`
- Clojure tree-sitter path: `parse-clojure-tree-sitter` и `parse-clojure-file` в `src/semidx/runtime/adapters.clj`
- Java tree-sitter path: `parse-java-tree-sitter` и `parse-java-file` в `src/semidx/runtime/adapters.clj`
- TypeScript tree-sitter path: `src/semidx/runtime/languages/typescript.clj`
- Elixir/Python/Lua entrypoints без tree-sitter: `src/semidx/runtime/languages/elixir.clj`, `src/semidx/runtime/languages/python.clj`, `src/semidx/runtime/languages/lua.clj`

## Трасса проверки

- Проверен список поддерживаемых языков в `runtime/language_activation.clj`
- Проверен runtime adapter layer на `tree-sitter`, `tree_sitter_enabled`, `*_engine`, `SCI_TREE_SITTER_*`
- Проверены parser entrypoints для `elixir`, `python`, `lua`, `typescript`
- Проверены тесты на наличие configured tree-sitter coverage для `clojure`, `java`, `typescript`

## Вывод

Эта заметка фиксирует coverage snapshot, а не окончательный priority order.

Итоговый cross-language ranking теперь зафиксирован в `ADR-028`:

- `elixir` — следующий приоритет для tree-sitter
- `python` — следующий кандидат на structural parser deepening, но не обязательно через tree-sitter
- `lua` — остается на bounded regex-first MVP, пока scope не расширится
