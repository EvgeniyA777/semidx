# Working Note: Phase 4 Runtime Hardening Complete

Краткая dated-заметка о том, что после последних runtime changes основной продуктовый Phase 4 больше не считается открытым.

## Что изменилось

- HTTP и gRPC теперь используют тот же normalized usage-event shape, что и library/MCP.
- Tenant/correlation context протянут сквозно через runtime edges:
  - `x-tenant-id`
  - `x-trace-id`
  - `x-request-id`
  - `x-session-id`
  - `x-task-id`
  - `x-actor-id`
- HTTP возвращает effective correlation обратно как `x-sci-*` response headers.
- gRPC возвращает correlation markers как `x-sci-*` trailers на error path.
- `SCI_USAGE_METRICS_JDBC_URL` теперь покрывает не только MCP, но и HTTP/gRPC service edges.

## Статус roadmap после этого

- Phase 2: done
- Phase 3: materially advanced, but still partial for compiler-grade depth
- Phase 4: done for the main roadmap scope
- Phase 5: next major tranche
